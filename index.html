Self-Reflection

Throughout the software engineering program at Southern New Hampshire University, I have learned a lot about computer science. I have learned how to communicate about the Software Development Life Cycle (SDLC), how to work in a team environment, how data structures and algorithms work, and how to create secure code.

For starters, I enrolled in a course titled Collaboration and Team Project. This course instructed me how to use Git to submit new branches on an existing file. Every student in our course was tasked with inserting our own favorite artists and songs into a “jukebox” java application. At the end of the course, we had one file that included all of our favorite songs and artists.

Another important course that I was enrolled in would be Software Development Lifecycle. Obviously, this course dealt with learning about the SDLC. I also learned a lot about working with the Agile method and utilizing scrum methods for sprint planning.

When it comes to data structures and algorithms, the course that helped me the most was my Data Structures and Algorithms course. In this course, we learned about various algorithms and what kind of data structures they can be used in. Our final project also involved incorporating one of the most useful data structure and algorithm combinations: the binary search tree.

As I progressed along my program, I enrolled in a Structured Database course. This course was extremely useful in teaching me how databases work and how they can be used. I also learned how to use SQL queries to create a database and a few tables. SQL became a language that I picked up on very easily, and I thoroughly enjoyed this course.

Lastly, in the term before my final term, I enrolled in a Secure Coding course. This course taught me the fundamentals of various security measures in code. We learned about buffer overflows, SQL injections, exceptions, encryption, and how to write a sound security policy. We also learned about the different levels of severity for security issues as well as the likelihood for them to occur.

Despite my lack of professional experience, I feel confident that I can adapt to any work environment that I may employed into. SNHU has taught me many valuable skills that can be utilized in a professional environment.

In my ePortfolio, I have included three artifacts: One that displays my knowledge on software design and engineering, one that displays my knowledge on data structures and algorithms, and one that displays my ability to work with a database. All three of these artifacts have been re-worked and enhanced to better showcase my abilities as a software developer.

Project 2 - Binary Search Tree

Binary Search Tree Link:

Binary Search Tree

This artifact is my final project from my Data Structures and Algorithms course. This project utilizes a binary search tree, one of the most efficient data structures and algorithms, to load, search, and delete bids from a CSV file. This project was originally created in April of 2021.

This artifact, as mentioned above, utilizes a binary search tree. A binary search tree takes a list of elements and sets a “half-point”. When a user searches for something, the algorithm will compare what they’re searching to the half-point. If it is larger, the algorithm will ignore the smaller half, then set a new half point between the original and the maximum (the opposite if it’s smaller). This is continued until either the item is found or if it does not exist. This is much more efficient than having the algorithm search through every single item one after another. This is why binary search trees are so efficient, and why I think this artifact is a great representation of my knowledge on data structures and algorithms.

I believe that I met the course objectives I planned to meet in this enhancement. I have great structure in my code, and I successfully showcase the layout of a binary search tree.

I, in the same case as milestone two, learned a lot while working on this project. I, obviously, have taken the Data Structures and Algorithms course in the past, however it has been over a year ago. So, it was nice to go and re-hash on some old projects to make it fresh in my memory. Honestly, I did not know entirely what I was doing when I worked on the artifact the first time, however, now, I can look back at the code and really understand what it means and how it works.
